# 突発単語会話 - 仕様書

## 概要

パーティーゲーム型Webアプリ。参加者全員に同じ単語が配られ、その単語を使って「目標」に最も合う文章を作成し、投票で勝者を決める。アイスブレイクや暇つぶしに最適な、知り合い同士で楽しむ対人個人戦ゲーム。

## ペルソナ / ユースケース

| 項目 | 内容 |
|------|------|
| 誰と | 友達、現実で会った初対面の人 |
| 状態 | 隣にいる / 通話中 / 会話手段がない(あると望ましい) |
| 目的 | アイスブレイク、暇つぶし、二次会、パーティーゲーム |
| 前提 | 知っている人とやる前提(匿名だと一期一会になり交流に繋がらない) |

## ゲームフロー

### 全体の流れ

1. **ルーム参加** - 合言葉を入力してルームに参加
2. **ゲーム開始** - 全員が開始ボタンを押せる（ホストの概念なし）
3. 以下を 3~4ラウンド 繰り返す:
   1. **目標提示** - お題が表示される (例: 「世界で一番食べたくないもの」)
   2. **単語決定フェーズ** - 各参加者が単語を入力し、プールから単語が選ばれる
   3. **文章作成フェーズ** - 配られた単語を使って文章を作成
   4. **投票フェーズ** - 全員の文章を見て、目標に最も合うものに投票
4. **リザルト** - 累計スコアで結果発表

### 単語フェーズ詳細

- 各参加者が単語を1つ提出する（**ユーザー入力 4個**）
- **ランダムプール 6個** をシステムが追加（単語リストは別途用意）
- 合計 **10個** の単語プールから全員に **同じ単語** が配られる

### 文章作成フェーズ詳細

- 文章は**自由入力**（テキストエリアに直接入力、クリックで挿入する方式ではない）
- 配られた単語のうち、**使用必須の数**と**文字数上限**は**常に固定**
  - 例: 4個の単語を全て使い、18字以内で文章を作る
- 入力された文章から**正規表現**で必須単語の使用を判定する
- 参加者自身が出した単語もそのまま使用可能

### 投票フェーズ詳細

- 全員の文章が一覧表示される
- 投票で目標に最も近い文章を選ぶ（**自分への投票は不可**）

## ルール詳細（補足）

| 項目 | 内容 |
|------|------|
| お題の出典 | 固定リスト |
| 単語プール内訳 | ユーザー入力 4個 + ランダムプール 6個（単語リストは別途用意） |
| 使用必須数・文字数上限 | 常に固定 |
| 文章入力方式 | 自由入力（クリック挿入ではない） |
| 単語含有判定 | 入力文章を正規表現でチェック |
| 自分への投票 | 不可 |
| 同点時の表示順 | 入室順（適宜） |

## 画面構成

| 画面 | パス | 説明 |
|------|------|------|
| 合言葉入力 | `/` (index.html) | 合言葉と表示名を入力してルームに参加 |
| 開始待ち | start.html | 参加者一覧の表示、全員がゲーム開始ボタンを押せる |
| 単語決定 | word.html | 目標表示、単語入力 |
| 文章作成 | sentence.html | 目標表示、単語表示、文章入力 |
| 投票 | vote.html | 全文章一覧、投票ボタン |
| リザルト | result.html | スコア表示、最初に戻るボタン |

## 技術選定

### フロントエンド

- **技術:** HTML / JavaScript(ES Modules) / CSS
- **構成:** 単一リポジトリ内、画面ごとにHTMLファイルを分割
- **デプロイ:** GitHub Pages
- **担当:** しげ、keigo

### バックエンド

- **言語:** Java 21
- **フレームワーク:** Spring Boot 3.2.5
- **DB:** PostgreSQL (ローカル: Docker / 本番: Neon)
- **DBアクセス:** JdbcTemplate
- **担当:** hata、jintan

### 通信方式

- **REST通信** (WebSocketは使用しない)
- **ポーリング** で状態変化を検知
- ユーザー識別は `X-User-Id` ヘッダ (クライアントでランダム文字列を生成)

## API設計

### ユーザー

| メソッド | パス | 説明 |
|---------|------|------|
| POST | `/users/{name}` | ユーザー名を登録 |

### ルーム

| メソッド | パス | 説明 |
|---------|------|------|
| POST | `/rooms/{passphrase}` | ルームに参加 (200: 成功 / 400: 作成済み) |
| PUT | `/rooms/{passphrase}` | ゲーム開始 (status = "STARTING") |
| GET | `/rooms/{passphrase}` | ルーム状態取得 (ポーリング用)。status, round, members, goal, distributedWords を返す |

### 単語

| メソッド | パス | 説明 |
|---------|------|------|
| POST | `/words/{passphrase}` | 単語を投稿 (body: `{ "value": "..." }`) |
| GET | `/words/{passphrase}` | 単語一覧を取得 |

### 文章

| メソッド | パス | 説明 |
|---------|------|------|
| POST | `/sentences/{passphrase}` | 文章を投稿 (body: `{ "value": "..." }`) |

### 共通仕様

- 全リクエストに `X-User-Id` ヘッダを付与
- User IDはクライアント側で生成しlocalStorageに保存
