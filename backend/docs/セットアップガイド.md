# バックエンド セットアップガイド

## 前提条件

- **Java 21** 以上（mise 推奨: `backend/.tool-versions` 参照）
- **Docker** と **Docker Compose**
- **Maven**（または `./mvnw` を使用）

---

## 1. ローカル DB の起動

```bash
cd backend
docker compose up -d
```

PostgreSQL が `localhost:5433` で起動します。（5432 が他プロセスで使用中の場合の回避）

### 接続情報（ローカル）

| 項目 | 値 |
|------|-----|
| Host | localhost |
| Port | 5433 |
| Database | app |
| User | app |
| Password | app |

---

## 2. アプリケーションの起動

```bash
./mvnw spring-boot:run
```

または、`application-local` プロファイルを明示する場合:

```bash
./mvnw spring-boot:run -Dspring-boot.run.profiles=local
```

---

## 3. 動作確認

- **ヘルスチェック**: http://localhost:8080/actuator/health
- **サンプル API**: http://localhost:8080/api/health

---

## 4. Neon に接続する場合

1. [Neon](https://neon.tech) でプロジェクトを作成
2. 接続文字列を取得
3. 環境変数または `application-neon.yml` を設定:

```yaml
spring:
  datasource:
    url: jdbc:postgresql://<host>/<db>?sslmode=require
    username: ${NEON_USER}
    password: ${NEON_PASSWORD}
```

起動時:

```bash
./mvnw spring-boot:run -Dspring-boot.run.profiles=neon
```

---

## 5. よくあるトラブル

### ポート 5432 が使用中

別の PostgreSQL が動いている場合、本プロジェクトはデフォルトで `5433` を使用しています。5433 も使用中の場合は `docker-compose.yml` のポートを `5434` などに変更し、`application.yml` の `url` も合わせて変更してください。

### Java バージョンが合わない

```bash
java -version
```

21 以上であることを確認。mise 利用時は `cd backend && mise install` で自動インストール。
